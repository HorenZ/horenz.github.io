Git教程

1.创建版本库

$ git init

1.1用命令git add告诉Git，把文件添加到仓库：

$ git add readme.txt  .-全部

1.2用命令git commit告诉Git，把文件提交到仓库：

$ git commit -m "本次提交说明"

1.3运行git status命令查看状态：

$ git status

git status命令可以让我们时刻掌握仓库当前的状态，上面的命令输出告诉我们，readme.txt被修改过了，但还没有准备提交的修改。

git diff顾名思义就是查看difference

如果git status告诉你有文件被修改过，用git diff可以查看修改内容

使用git reset命令回退版本

$ git reset --hard HEAD^

在Git中，用HEAD表示当前版本

上一个版本就是HEAD^，上上一个版本就是HEAD^^

当然往上100个版本写100个^比较容易数不过来，所以写成HEAD~100

┌────┐
│HEAD│
└────┘
   │
   └──> ○ append GPL
        │
        ○ add distributed
        │
        ○ wrote a readme fil
        
┌────┐
│HEAD│
└────┘
   │
   │    ○ append GPL
   │    │
   └──> ○ add distributed
        │
        ○ wrote a readme file

用git log可以查看提交历史，以便确定要回退到哪个版本

要重返未来，用git reflog查看命令历史，以便确定要回到未来的哪个版本。

$ git reflog
e475afc HEAD@{1}: reset: moving to HEAD^
1094adb (HEAD -> master) HEAD@{2}: commit: append GPL
e475afc HEAD@{3}: commit: add distributed
eaadf4e HEAD@{4}: commit (initial): wrote a readme file

Git允许我们在版本的历史之间穿梭，使用命令git reset --hard commit_id

$ git reset --hard e475afc

版本库（Repository）

每次修改，如果不用git add到暂存区，那就不会加入到commit中

撤销修改

git checkout -- file可以丢弃工作区的修改

暂存区无内容，则撤回到版本库一样的状态

暂存区有内容，撤回到暂存区状态

用命令git reset HEAD 可以把暂存区的修改撤销掉（unstage），重新放回工作区

将add的内容从暂存区取出

Git知道你删除了文件，因此，工作区和版本库就不一致了，git status命令会立刻告诉你哪些文件被删除了

现在你有两个选择，一是确实要从版本库中删除该文件，那就用命令git rm删掉，并且git commit，文件就从版本库中被删除了

将本地库所有内容推送到远程库上：

git remote add origin git@github.com:michaelliao/learngit.git
